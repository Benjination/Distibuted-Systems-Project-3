\section{Q3 — Raft: Leader Election}

\subsection*{Design}

Leader election is implemented in \texttt{node2\_api\_server/raft\_node.py}
and defined in \texttt{proto/raft.proto}. It runs as a background thread
on \textbf{all 5 nodes} alongside the existing gRPC services.

\textbf{Timeout settings (as required):}
\begin{itemize}
  \item Heartbeat timeout: \textbf{1 second} (fixed, sent by the leader)
  \item Election timeout: \textbf{random in [1.5s, 3.0s]} (chosen independently per node at startup and on each reset)
\end{itemize}

\subsection*{State Machine}

All nodes start as \textbf{Follower}. The election cycle is:

\begin{enumerate}
  \item \textbf{Follower} — resets its election timer on each \texttt{AppendEntries} heartbeat received.
        If the timer expires with no heartbeat, it transitions to Candidate.
  \item \textbf{Candidate} — increments its term, votes for itself, sends \texttt{RequestVote}
        to all other nodes. Becomes Leader on majority ($\geq 3$ of 5 votes).
        Reverts to Follower if it hears from a legitimate leader (higher term).
  \item \textbf{Leader} — sends \texttt{AppendEntries} heartbeats to all followers every 1 second
        to prevent new elections.
\end{enumerate}

\subsection*{Proto Definition}

\begin{lstlisting}[language={}]
service RaftService {
  rpc RequestVote(RequestVoteMsg)     returns (RequestVoteReply);
  rpc AppendEntries(AppendEntriesMsg) returns (AppendEntriesReply);
}
\end{lstlisting}

\subsection*{Required Print Output}

\begin{lstlisting}[language={}]

Node 1 runs RPC RequestVote called by Node 3
Node 2 sends RPC AppendEntries to Node 4
Node 4 runs RPC AppendEntries called by Node 2
\end{lstlisting}
