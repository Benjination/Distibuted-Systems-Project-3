\section{Project Overview}

This project extends the \textbf{pharmacy\_system} distributed inventory system
(originally developed for Project 2) with two distributed coordination protocols:
Two-Phase Commit (2PC) for atomic stock updates, and a simplified version of the
Raft consensus algorithm for leader election and log replication.

The base system is a gRPC-based pharmacy inventory service backed by a PostgreSQL
database. For this project it was scaled from 2 to \textbf{5 API server nodes},
all communicating over a shared Docker bridge network (\texttt{pharmacy-net}).
