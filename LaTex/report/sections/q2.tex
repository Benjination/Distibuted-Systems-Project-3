\section{Q2 — Two-Phase Commit: Decision Phase}

\subsection*{Design}

After collecting all votes, the coordinator sends either \texttt{GlobalCommit}
or \texttt{GlobalAbort} to all 5 participants. Both RPCs belong to the same
\texttt{TwoPhaseParticipantService} defined in Q1, fulfilling the requirement
that both phases share a single proto file.

The \texttt{CoordinatorService} is exposed on port \texttt{50053} on Node 1 only.
A client calls \texttt{UpdateStock2PC} which triggers the full Q1 + Q2 protocol.

\subsection*{Decision Logic}

\begin{itemize}
  \item \textbf{All voted COMMIT} $\Rightarrow$ coordinator sends \texttt{GlobalCommit} to all 5 nodes.
        Node 1 (coordinator node) executes the actual \texttt{UPDATE drugs SET quantity = ...}
        SQL write. The other 4 nodes log a local commit without touching the database,
        since all nodes share a single PostgreSQL primary instance.
  \item \textbf{Any voted ABORT} $\Rightarrow$ coordinator sends \texttt{GlobalAbort} to all 5 nodes.
        Each participant discards the pending transaction. No database write occurs.
\end{itemize}

\subsection*{Required Print Output — Commit Path}

\begin{lstlisting}[language={}]
Phase decision of Node 1 sends RPC GlobalCommit to Phase voting of Node 2
Phase voting of Node 2 receives RPC GlobalCommit from Phase decision of Node 1
Phase voting of Node 2 sends RPC GlobalCommitAck to Phase decision of Node 1
\end{lstlisting}

\subsection*{Required Print Output — Abort Path}

\begin{lstlisting}[language={}]
Phase decision of Node 1 sends RPC GlobalAbort to Phase voting of Node 2
Phase voting of Node 2 receives RPC GlobalAbort from Phase decision of Node 1
Phase voting of Node 2 sends RPC GlobalAbortAck to Phase decision of Node 1
\end{lstlisting}
