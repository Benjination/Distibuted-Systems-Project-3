CURRENT STATE OF pharmacy_system
---------------------------------
- node1: NGINX load balancer (port 8080) — routes to api-server-a and api-server-b
- node2: API Server A (Python gRPC, port 50051) — handles AddDrug, GetDrug, UpdateStock,
         DeleteDrug, ListDrugs, GetLowStock; writes directly to db-primary
- node3: API Server B (same image as A, port 50051) — identical to A
- node4: PostgreSQL Primary DB
- node5: PostgreSQL Replica DB (streaming replication from primary)
- node6: pgAdmin (monitoring)
- monolith-rest-api + mono-db (comparison REST app, can largely be ignored)

All nodes are in Python, containerized, and communicate over a Docker bridge network.
The pharmacy.proto is already defined and compiled inside the Dockerfile at build time.
server.py is clean and straightforward — all RPCs are ~10 lines each.
